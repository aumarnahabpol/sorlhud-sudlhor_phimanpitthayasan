# 💡 Idea Canvas — Lean Canvas (4 ช่องแรก)

> ทำช่วงท้ายวัน เปลี่ยน "ของที่รันได้วันนี้" → "ไอเดียนวัตกรรมที่ต่อยอด Day 2 ได้"
> เริ่มจาก **ปัญหา** ไม่ใช่ "เรามี sensor อะไร" · ทำ 4 ช่องนี้ให้คมก่อน ที่เหลือไว้ทีหลัง

- **ชื่อ Prototype (ชั่วคราว):** AI ตรวจจับการใส่หมวกนิรภัยเพื่อการขับขี่ที่ปลอดภัย

---

## 1️⃣ Problem — ปัญหาคืออะไร

เขียน 2–3 ข้อ ที่มีคน "ลำบาก / เสียเวลา / พลาด" จริง:
```
- ปัญหาการขับขี่บนท้องถนนโดยไม่สวมหมวกนิรภัย ทำให้เกิดอุบัติเหตุร้ายแรงได้ หากไม่ป้องกันศีรษะ
- ตำรวจไม่สามารถตรวจตราทุกจุดบนท้องถนนได้
- ประชาชนบางพื้นที่เมินเฉยต่อกฎจราจรเพราะความเข้มงวดยังไม่มากพอ
```
**ตอนนี้คนแก้ปัญหานี้ยังไง (ของเดิม)?** ใช้ตำรวจจัดการดูแล ตั้งด่าน เพื่อจับคนไม่ใส่หมวกนิรภัย

## 2️⃣ Customer — ใครคือผู้ใช้จริง

เจาะให้ชัด ไม่ใช่ "ทุกคน":
```
 หน่วยงานที่ดูแลและควบคุมเรื่องการขับขี่ปลอดภัยบนท้องถนน
```

## 3️⃣ Value (USP) — คุณค่าเด่นในประโยคเดียว

```
"[ชื่อของเรา] ช่วย [ผู้ใช้] [ทำอะไรได้]
 ต่างจาก [ของเดิม] เพราะ [ข้อได้เปรียบ]"
AI ตรวจจับการใส่หมวกนิรภัยเพื่อการขับขี่ที่ปลอดภัย ช่วยตรวจจับคนไม่ใส่หมวกผ่านกล้องที่ฝัง AI ได้ทันที ต่างจากการใช้จำนวนตำรวจจราจรที่ต้องใช้หลายคน เพราะ เราสามารถติดตั้งกล้อง AI จับการสวมหมวกนิรภัยได้หลายจุดโดยไม่ต้องมีคนควบคุมในทุกพื้นที่
```
ตัวอย่าง: *"SafeFall ช่วยผู้สูงอายุที่อยู่คนเดียว ขอความช่วยเหลือได้ทันเมื่อล้ม — ต่างจาก smartwatch แพงๆ เพราะใช้อุปกรณ์ราคาถึงได้และไม่ต้องต่อเน็ต"*

## 4️⃣ Solution — แก้ด้วยอะไร

- **Input ที่ใช้:** กล้อง
- **Class ที่ AI แยก:** รูปคนสวมหมวกนิรภัย รูปคนไม่สวมหมวกนิรภัย
- **Output ตอบสนอง:** buzzer

---

## 🎯 MVP — ฟังก์ชันเดียวที่ต้องสาธิตให้ได้

```
Input → AI Model → UNO Q → Prediction → Output ที่ผู้ใช้เห็น
รับภาพจากกล้อง -> Model ตรวจจับคนและแยกว่าคนสวมหมวกหรือไม่ -> UNO Q -> Classification -> ส่งเสียงเมื่อพบคนไม่สวมหมวกนิรภัย
```

**1 core function ที่จะโชว์ Day 2/3:** โมเดลแยกได้ว่ารูปไหนสวมหมวกหรือไม่สวมหมวก
*(เช่น "ตรวจได้ว่า Safe/Warning แล้วไฟ+เสียงเตือน")*

> คาถา: **Build less, prove more** — ฟังก์ชันเดียวที่ทำงานจริง + เล่าได้ ดีกว่าหลายอย่างที่ทำไม่สุด
